An application for a bag brand, which is my first full-stack app through which i learnt best practices for scalable web development.

## üìã Project Overview

This project demonstrates a server-side application with separate roles for **customers** and **owners**, each with distinct functionalities and access levels.

So, it will be helpful for those who want to understand the best practices in backend implementation or freshers want to test out the functionalities.

This is not a top notch UI based full Stack application; it is an app mostly concerned with the backend and it's proper and better implementation.

## üöÄ Features

### Authentication & Authorization

- Secure user signup and login
- Session-based authentication with cookies
- Role-based access control [Customers vs Owners]
- Password hashing and user data protection

### User Roles

- **Customers**: Browse products, manage cart, place orders, view order history, and update account data
- **Owners**: Admin dashboard, manage products, inventory control, order management

### Core Functionality

- Product catalog with detailed views
- Shopping cart and order processing
- User profile management
- Fully responsive design
- Complete Separation of concerns
- Proper data retrieval and storage
- cloud-based storage
- protected routes
- proper environment variable setup
- Coded for the Production
